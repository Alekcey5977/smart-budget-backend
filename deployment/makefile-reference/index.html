    Документация / Деплой / Makefile

    Справочник Makefile¶

    Все команды выполняются из корня репозитория: make <команда>

    Управление сервисами¶

    Команда Описание
    make start Запустить весь стек (docker compose up -d)
    make stop Остановить контейнеры (данные сохраняются)
    make restart Перезапустить все контейнеры
    make down Остановить и удалить контейнеры (volumes сохраняются)
    make build Пересобрать Docker-образы сервисов
    make logs Показать логи всех сервисов в реальном времени
    make status Показать статус контейнеров (docker compose ps)

    Тестовые данные¶

    Команда Описание
    make generate-test-data Сгенерировать JSON-файлы тестовых данных (testData/*.json)
    make load-test-data Загрузить банки, счета и транзакции в pseudo-bank-service
    make load-test-images Загрузить аватарки и иконки в images-service

    Порядок выполнения: generate-test-data → load-test-data → load-test-images.

    Файлы после генерации: - testData/pseudo_bank_test_data.json — транзакции, мерчанты, категории - testData/images_data.json — данные изображений - testData/test_accounts_info.md — справочник тестовых счетов с хэшами

    Установка зависимостей¶

    Команда Описание
    make install Создать .venv и установить зависимости для всех 8 сервисов
    make clean-venvs Удалить все .venv директории

    Требуется Python 3.11–3.13. При отсутствии: brew install python@3.11.

    Тестирование¶

    Команда Описание
    make test Unit + интеграционные тесты для всех сервисов
    make test-unit Только unit тесты
    make test-e2e-start Поднять изолированный тестовый стек (порты 18000+)
    make test-e2e Запустить E2E тесты против тестового стека
    make test-e2e-stop Остановить и удалить тестовый стек (все данные удаляются)

    make test-e2e-start автоматически: 1. Поднимает docker-compose.test.yml 2. Ждёт готовности gateway (до 2 минут) 3. Патчит контейнеры с Prometheus инструментацией 4. Загружает тестовые данные

    Нагрузочное тестирование (k6)¶

    Требует: make test-e2e-start (или работающий стек), установленный k6.

    Команда VUs Длительность Цель
    make k6-smoke 1 30 сек Sanity check — все эндпоинты отвечают
    make k6-load 0→50→0 ~4 мин Реалистичная нагрузка
    make k6-stress 0→150→0 ~5 мин Поиск предела производительности
    make k6-spike 20→1000→20 ~1 мин Устойчивость к резкому всплеску
    make k6-max 0→3000 ~1 мин Абсолютный предел (сломать систему)
    make k6-high 0→5000 75 сек За пределом (только наблюдение)
    make k6-extreme 0→10000 90 сек Экстремальная нагрузка (только наблюдение)
    make k6 — ~10 мин Последовательно: smoke → load → stress

    Переопределение URL: 

    make k6-load K6_BASE_URL=http://localhost:8000   # против dev-стека

    Экспорт метрик в Prometheus: 

    make k6-load K6_PROMETHEUS_RW=http://localhost:19090/api/v1/write

    Обслуживание¶

    Команда Описание
    make clean Остановить и удалить всё включая volumes (данные удаляются, требует подтверждения)
    make reset-db Полный сброс всех БД — данные удаляются, ID начинаются с 1 (требует подтверждения)

    Переменные Makefile¶

    Переменная Default Описание
    K6_BASE_URL http://localhost:18000 URL для k6 тестов
    K6_PROMETHEUS_RW http://localhost:9090/api/v1/write Remote write URL Prometheus
    TEST_PROJECT smartbudget-test Имя Docker Compose проекта для тестов
