# Pseudo Bank Service

**Порт:** 8004 | **БД:** PostgreSQL :5436 (pseudo_bank_db)

Симулирует банковский API для разработки и тестирования. Не является частью продакшн-системы. Содержит 10 предзагруженных тестовых счетов с историей транзакций.

---

## Назначение

Заменяет реальный банк. Позволяет разрабатывать и тестировать функции синхронизации, добавления счетов и просмотра транзакций без интеграции с реальными финансовыми системами.

**Потребители сервиса:**
- `users-service` — валидация при добавлении счёта
- `transactions-service` — экспорт транзакций при синхронизации

## Хэширование счётов

Номера счетов никогда не передаются в открытом виде между сервисами. Вместо этого используется HMAC-SHA256:

```python
import hmac, hashlib

def hash_account(account_number: str, key: str) -> str:
    return hmac.new(
        key.encode("utf-8"),
        account_number.encode("utf-8"),
        hashlib.sha256
    ).hexdigest()

# Пример:
hash_account("40817810099910004312", "bank-account-secure-key-2026")
# → "0eb1e101dd9a63977ba423f0b31ae575373940dc86e51f4719e06203e96445b0"
```

Ключ задаётся в `BANK_SECRET_KEY`. Хэш используется как общий идентификатор между `users_db` и `transactions_db`.

## Эндпоинты

| Метод | Путь | Описание |
|-------|------|----------|
| `POST` | `/pseudo_bank/validate_account` | Проверить существование счёта по хэшу |
| `GET` | `/pseudo_bank/account/{hash}/export` | Экспорт данных счёта (query: `since=`) |
| `POST` | `/pseudo_bank/categories/bulk` | Загрузка справочника категорий |
| `POST` | `/pseudo_bank/mcc_categories/bulk` | Загрузка MCC-маппингов |
| `POST` | `/pseudo_bank/merchants/bulk` | Загрузка мерчантов |
| `POST` | `/pseudo_bank/banks/bulk` | Загрузка банков |
| `POST` | `/pseudo_bank/bank_accounts/bulk` | Загрузка тестовых счетов |
| `POST` | `/pseudo_bank/transactions/bulk` | Загрузка тестовых транзакций |
